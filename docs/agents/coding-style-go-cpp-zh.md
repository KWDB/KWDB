# Go 与 C++ 编码风格

本文为 [`coding-style-go-cpp.md`](coding-style-go-cpp.md) 的中文对照稿。

本仓库在开源场景下以该英文文件为规范性摘要。若与相邻模块代码或工具链明显冲突，**优先贴近当前子树惯例**，再通过评审统一文档与实现。

---

## Go

### 格式化

- 使用 `gofmt`；推荐使用 `goimports` 去未用 import 并稳定分组。
- 函数签名单行过长时拆分：名称、参数列表、返回列表分行；参数列表的收尾 `)` 与 `func` 缩进对齐；极长参数表可每行一个参数，返回类型列表同理。

### 声明分组

- 相关 `const`、`var`、`type` 放一起；互不相关不要硬凑。
- 包级与函数内均可出现分组。

### 行宽

- 代码约 100 列、注释约 80 列为目标（制表按 2 空格计），可读性受损时不必机械换行。

### 注释

- 导出符号需 `godoc` 风格：完整句子，以被说明的名字起句、句点结尾。
- 注释语种推荐英文，减少编码与工具链摩擦。
- `package` 前需包注释，一包一文件即可。
- 可用 `TODO:`、`FIXME:`、`NOTE:`，并保证可行动。

### 命名

- 需要旁白才能懂的名字应改名。
- 可检索优先；单字母仅限很小局部；名字长度与作用域相称。
- 避免无信息区分如 `Product` / `ProductInfo` / `ProductData`。
- **函数：** 小驼峰；动词或动宾；`get/set/is` 仅在与惯用读法一致时使用。
- **结构体：** 名词性；避免笼统的 `Manager`、`Processor`、`Data`、`Info`。
- **包名：** 全小写；尽量单词化；无下划线与 `MixedCaps`；一般不用复数；禁用 `common`、`util`、`shared`、`lib` 等无信息名词。
- **常量：** 全大写+下划线；枚举可起独立类型；常量太多时可用统一前缀。
- **变量：** 正常英文或公认缩写；布尔以 `Has`、`Is`、`Can`、`Allow` 等起头；专有词缩写在 Go 中遵守惯例如 `API`、`URL`、`ID`，避免 `Url`。
- `:=` 与 `var` 取舍：已显式赋值多用 `:=`；强调零值或类型时用 `var`。作用域越小越好，但勿牺牲清晰度。

### 函数与文件

- 函数大致按调用顺序；按接收者类型聚类方法。

### 结构体

- 声明与字面量初始化在多时多行更清晰。
- 取空结构地址时若本地风格普遍，可优先 `&T{}` 胜于 `new(T)`。

### 控制流

- `if` 短变量声明缩小作用域。
- 尽早 `return` 处理错误与特例，控制嵌套层级。
- `panic` 仅限不可恢复错误或被维护者明确允许的少数场景。

### 导入

- 标准库与非标准库空行分组；非测试禁用点导入；禁用 `./relative` 式相对路径；交给 `goimports` 维护顺序。

### 参数传递

- 小对象按值传；大结构体为降低成本可用指针。
- 不要给 map、slice、channel 再套指针。

### 面向 SQL 的代码

- 同一概念跨语句用词一致；SQL 侧标识倾向 `snake_case`；避免列为 `user` 等易与语法冲突的词；泛名用 `zone_id`、`job_id` 消歧；若以多种句柄（按名/按 ID）引用同一对象，命名中体现句柄类型。

### 测试

- `*_test.go`；`Test` 前缀；表驱动子测试；表字段区分输入与期望（如 `give`/`want`）。

### 静态检查

- 对改动包跑 `go vet`；其余以 CI 与项目约定为准。

## C++

总体版式与命名对齐公开 [Google C++ Style Guide](https://google.github.io/styleguide/cppguide.html)（若本文件或本地代码有更严规则则从其）。

### 常量与宏

- 多用 `const`/`constexpr` 与强类型枚举；少用无类型宏。
- 相关整型成组为枚举；仅在需要稳定数值映射时显式赋值；避免无意重复枚举取值；语义不同即使数值相同也应分常量；对参数与不改成员的方法加 `const`。

### 初始化与对象生存期

- 禁止对非 POD 用 `memset`/`memcpy` 糊弄 `this`（可破坏 vtable 等）。
- 靠首次使用点声明并初始化；构造函数保持简单，确需错误返回、异常、长流程、未完整 `this` 上虚调用等再用 `Init()` 型辅助。
- **成员初始化列表顺序须与类内声明顺序一致**；留意跨翻译单元静态初始化顺序，倾向受控初始化（单例工厂、`main`、`pthread_once` 等）。

### 类型转换与多态

- 用 C++ 风格 cast；慎 `reinterpret_cast`、`const_cast`；业务分支忌到处 `dynamic_cast`，优先虚函数体系。

### 函数

- `inline` 仅 1～10 行简单逻辑；虚函数、递归、大函数不宜内联；大块内联体放 `*.inl`。
- 入参能用 `const&` 且无空指针语义时优先后者；未用形参在重写中可删名或注释占位；默认参数少用；模块边界少依赖函数指针。

### 类与 OOP

- 类宜小且职责单一；数据成员私有化，避免返回可写内部引用/指针暴露实现；接口类避免把保护/私有细节摊给无关方导致重编；需要稳定 ABI 或加速编译可用 PIMPL。
- RAII 资源类要自管或 `= delete` 拷贝/移动/赋值；`operator=` 防自赋值且返回 `*`；单参数构造 `explicit`；多态基类析构需 `virtual`；构造/析构中慎用虚调用。
- 倾向组合胜实现继承；继承层次浅；慎用多实现继承（多纯接口稍可接受）；`public` 表 is-a；不要在虚函数重写再带默认参数；不要用不同参数表在派生链上「藏」重载意图。

### 运算符

- 语义须贴近内置直觉；与其堆微妙重载不如显式命名函数；C 边界上避免重载困扰 C 侧。

### 命名空间与作用域

- 用命名空间归类；头文件忌 `using namespace` 污染；嵌套类与局部类少用；无名命名空间或文件内 `static` 限制实现细节。

### 全局与静态

- 非 POD 全局/静态生命周期顺序易坑，应限制或明确引导初始化。

### 模板

- 有意使用；注意编译膨胀与诊断难求；复杂模板参数用 `const` 引用或指针传；在声明处写清约束；不稳的模板接口慎跨共享库 ABI 暴露。

### 与 C 链结

- `extern "C"` 内只放安全声明，避免奇怪地塞入层层 include。

### 友元与 RTTI

- 友元少用；优先虚函数或访问者等模式，少靠到处 `typeid`/`dynamic_cast`。

### `sizeof`

- 写 `sizeof var` 以利于类型重构。

### 资源管理

- `new`/`new[]` 与 `delete`/`delete[]` 配对；所有权「谁申请谁释放」除非有转让文档；RAII/智能指针优先；`delete` 后按规范置空；嵌套结构自底向上释放。

### 线程与并发

- 可变共享状态须加锁并把临界区做小；多锁固定顺序；避免持锁调用未知可能回锁代码；用 RAII 锁防早退忘释放；跨进程协调优先消息、Socket、带互斥的共享内存或事务存储，慎共享文件；可重入助手多用栈上数据少碰隐藏全局。

### 异常（若模块启用）

- 少用并明确失败路径；构造/析构一般不放业务异常；按值抛、`const&` 收；勿让异常穿越无约定的模块边界。

### 错误策略

- 各模块认清「异常 vs 返回码」契约，在边界做翻译；事务场景考虑基本保证/强保证/不抛承诺。

### 标准库习惯

- 不要长期保存 `std::string::c_str()`；`shared_ptr` 创建优先 `make_shared`；同一对象不要混用裸拥有指针与共享指针；环用 `weak_ptr`；禁用 `auto_ptr`；不要拿 `unique_ptr`/`shared_ptr` 管数组。

### 容器与迭代器

- 注意失效规则与复杂度；`empty()` 与 `size()` 选择依实现史与性能谨慎。

### 性能

- 先测后改；容器与算法匹配访问模式；能用初始化列表就不必默认构造后再赋值；小心隐式临时、后置 `++` 于迭代器、iostream 同步瓶颈等。

### 命名与排版（KWDB）

- 类名与大写驼峰的自由函数；数据成员小写 snake + 尾随下划线 `_`。
- 类内建议顺序：`public`/`protected`/`private` 段；每段内 typedef/枚举/常量→构造→析构→方法（含 static）→数据（含 static）。
- `//` 注释，斜杠后空格；待发代码 TODO 须有责任人/日期等政策要求。

### 可移植性

- 不写死 `int/long` 宽度；`<cstdint>` 或工程 typedef；指针与整数转换用 `intptr_t`/`uintptr_t`；磁盘/跨 CPU 的结构注意对齐与端序；网络字节序显式转换；有符号无符号宽窄转换要谨慎；64 位字面量 `LL`/`ULL`；LP64 上 `sizeof(void*)` 未必等于 `sizeof(int)`；编译器专有指令应收进条件宏。

### 国际化

- 明确 UTF-8 `std::string` 与宽字符 API 选型；字节缓冲按 UTF-8 最大膨胀预留。

### 性能论证

- 微优化要结合公开技术报告与本仓库剖析数据，忌凭印象。
